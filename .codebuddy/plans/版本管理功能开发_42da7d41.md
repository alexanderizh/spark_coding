## 产品概述

在 admin 管理后台新增【版本管理】功能页面，用于管理 app 和 desktop 两个端的版本发布信息。

## 核心功能

- **双 Tab 切换**：app 版本（安卓）、desktop 桌面端版本
- **字段定义**：id（自动生成）、版本号（手输）、平台（app：安卓；desktop：macos/windows）、下载链接（手输）、发布说明（可选）
- **CRUD 操作**：支持新增、查看列表、编辑、删除版本记录
- **数据存储**：统一存储在同一数据表，通过 type 字段区分类型

## 技术栈

- **前端**：React 18 + Ant Design 5 + Less + Vite（与现有 admin 项目一致）
- **后端**：MidwayJS 3 + TypeORM 0.3 + MySQL（与现有 server 项目一致）
- **认证**：复用现有 Admin Basic Auth 机制

## 实现方案

### 架构设计

遵循现有项目的分层架构模式：

```
前端页面 → API Client → 后端 Controller → Service → Entity → MySQL
```

### 数据模型设计

创建统一的 `versions` 数据表，通过 `type` 字段区分版本类型：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| id | UUID | 主键，自动生成 |
| type | varchar(16) | 版本类型：app / desktop |
| version | varchar(32) | 版本号，如 1.0.0 |
| platform | varchar(16) | 平台：android / macos / windows |
| download_url | varchar(512) | 下载链接 |
| release_notes | text | 发布说明（可选） |
| created_at | datetime | 创建时间 |
| updated_at | datetime | 更新时间 |


### API 设计

复用现有 `/api/admin` 路由前缀和认证中间件：

- `GET /api/admin/versions` - 获取版本列表（支持 type 筛选、分页）
- `POST /api/admin/versions` - 创建版本
- `PUT /api/admin/versions/:id` - 更新版本
- `DELETE /api/admin/versions/:id` - 删除版本

## 实现要点

- **前端**：使用 Ant Design 的 Tabs 组件切换类型，Modal + Form 实现新增/编辑弹窗，Table 展示列表
- **后端**：参考现有 Session 实体和 Service 的实现模式，保持代码风格一致
- **认证**：复用 AdminAuthMiddleware，无需额外配置

## 设计风格

遵循现有 admin 项目的 Ant Design 默认主题，保持与「概览」「会话列表」页面一致的视觉风格。

## 页面布局

- **顶部**：页面标题 + 类型 Tab 切换（app 版本 / desktop 版本）+ 新增按钮
- **中部**：数据表格（版本号、平台、下载链接、发布说明、创建时间、操作列）
- **弹窗**：新增/编辑表单弹窗（版本号、平台下拉、下载链接、发布说明文本域）

## 交互设计

- Tab 切换时自动筛选对应类型的版本列表
- 平台字段根据当前 Tab 显示不同选项（app Tab：安卓；desktop Tab：macOS、Windows）
- 操作列包含「编辑」「删除」按钮，删除需二次确认
